## 3. 登录方式

### 3.1 短信验证码登录

| # | 字段 | 类型 | 规则 |
|---|------|------|------|
| 1 | 手机号 | 手机号 | 必填，11位 |
| 2 | 验证码 | 文本 | 必填，6位数字 |

**验证码规则：**
- 倒计时60s
- 短信模板：[XXX]验证码为XXXX，5分钟内有效
- 错误3次后需图形验证码

### 3.2 密码登录（如有）

| # | 字段 | 类型 | 规则 |
|---|------|------|------|
| 1 | 手机号/账号 | 文本 | 必填 |
| 2 | 密码 | 密码框 | 必填，6-20位 |

### 3.3 第三方登录

| 平台 | 说明 |
|------|------|
| 微信 | 静默获取openid，首次登录引导绑定手机号 |
| Apple | iOS专属 |
| QQ | Web/H5专属 |

### 3.4 一键登录（小程序）

| 方式 | 说明 |
|------|------|
| 手机号快登 | button open-type="getPhoneNumber" |
| 微信授权 | 静默获取昵称头像，首次需绑定手机 |

## 4. 注册字段

| # | 字段 | 类型 | 规则 |
|---|------|------|------|
| 1 | 手机号 | 手机号 | 必填，11位 |
| 2 | 验证码 | 文本 | 必填，6位数字 |
| 3 | 密码 | 密码框 | 必填，6-20位，字母+数字组合 |
| 4 | 确认密码 | 密码框 | 必填，与密码一致 |

---

## 5. 校验规则

| 字段 | 校验时机 | 错误提示 |
|------|----------|----------|
| 手机号 | 失去焦点 | 手机号格式不正确 |
| 验证码 | 提交时 | 验证码不能为空 / 验证码错误 |
| 密码 | 失去焦点 | 密码格式不正确（6-20位字母+数字） |
| 确认密码 | 失去焦点 | 两次密码输入不一致 |

---

## 6. 登录/注册后流程

| 步骤 | 说明 |
|------|------|
| 点击登录/注册 | 校验 → 调接口 |
| 成功 | 存储Token → 跳转目标页（来源页面或首页） |
| 失败 | 提示错误信息 |
| 第三方首次登录 | 跳转绑定手机号页 |

---

## 7. 第三方登录绑定逻辑

```
第三方登录 → 判断是否已绑定账号
  ├── 已绑定 → 直接登录成功
  └── 未绑定 → 跳转「绑定手机号」页
                 ├── 绑定成功 → 登录成功
                 └── 绑定失败 → 提示并留在绑定页
```

## 8. 接口契约

### 8.1 发送验证码

**接口路径：** `POST /api/sms/send`

```json
{
  "phone": "string",
  "type": "login | register | bind"
}
```

### 8.2 验证码登录

**接口路径：** `POST /api/auth/login`

```json
{
  "phone": "string",
  "code": "string"
}
```

**响应：**
```json
{
  "code": 0,
  "data": {
    "token": "string",
    "userInfo": {
      "id": "string",
      "nickname": "string",
      "avatar": "string"
    }
  }
}
```

### 8.3 第三方登录

**接口路径：** `POST /api/auth/third-party`

```json
{
  "platform": "wechat | apple | qq",
  "openid": "string"
}
```

## 10. 端适配说明

### 小程序
- 手机号获取：使用 button open-type="getPhoneNumber"
- 微信登录：wx.login()
- 登录状态：Storage存token

### H5
- 微信登录：redirect跳转授权页
- 回调处理：URL携带code参数
- 协议勾选：checkbox

### Web
- 密码框：支持显示/隐藏切换
- 回车登录：Enter键触发表单提交
- 记住登录态：checkbox
