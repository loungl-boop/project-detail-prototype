## 3. 表单字段

### 3.1 基本信息

| # | 字段名 | 字段key | 类型 | 必填 | 默认值 | 规则/说明 |
|---|--------|---------|------|------|--------|-----------|
| 1 | 名称 | name | 文本输入 | ✓ | - | 2-50字符，不允许重复 |
| 2 | 类型 | type | 下拉选择 | ✓ | 类型A | 选项：类型A/类型B/类型C |
| 3 | 状态 | status | 开关 | ✓ | 启用 | 启用 / 禁用 |
| 4 | 负责人 | owner | 人员选择器 | ✓ | 当前用户 | 只允许选择一人 |
| 5 | 生效日期 | startDate | 日期选择 | ✓ | 今天 | 不可选过去日期 |

### 3.2 扩展信息

| # | 字段名 | 字段key | 类型 | 必填 | 规则/说明 |
|---|--------|---------|------|------|-----------|
| 6 | 备注 | remark | 多行文本 | - | 最多500字符 |
| 7 | 附件 | attachments | 文件上传 | - | 支持jpg/png/pdf，单文件≤10MB，最多5个 |
| 8 | 关联订单 | orders | 关联选择器 | - | 弹窗选择，可多选 |

## 5. 表单校验规则

| 字段 | 校验类型 | 规则 |
|------|----------|------|
| name | 非空 | 不能为空 |
| name | 字符长度 | 2-50字符 |
| name | 唯一性 | 提交时校验，同名已存在则提示 |
| type | 非空 | 不能为空 |
| startDate | 范围 | 不能早于今天 |
| remark | 字符长度 | 最多500字符 |
| attachments | 文件类型 | 仅支持 jpg/png/pdf |
| attachments | 文件大小 | 单文件≤10MB |

**校验失败提示：**
- 输入框下方红色文字，如：`名称不能为空`
- 提交时如有错误，滚动到第一个错误处并高亮

## 6. 交互说明

| 交互点 | 行为 | 结果 |
|--------|------|------|
| 点击「保存」 | 校验全部字段 | 成功 → 跳转列表页；失败 → 高亮错误字段 |
| 点击「取消」 | - | 弹窗确认：未保存的内容将丢失，确定离开？ |
| 页面离开（未保存） | - | 弹窗确认（同上） |
| 点击附件「上传」 | 打开文件选择器 | 上传成功 → 显示缩略图/文件名 |
| 点击附件「删除」 | 删除已上传文件 | 确认后移除 |

## 7. 接口契约

### 7.1 详情查询（编辑时）

**接口路径：** `GET /api/{module}/:id`

> 编辑页进入时调用，填充表单默认值

### 7.2 保存（新增/编辑）

**接口路径：** `POST /api/{module}/save`（新增）  
**接口路径：** `PUT /api/{module}/:id`（编辑）

**请求参数：**

```json
{
  "name": "string",        // 必填
  "type": "string",        // 必填，选项：TYPE_A / TYPE_B / TYPE_C
  "status": "number",       // 必填，1-启用 2-禁用
  "owner": "string",       // 必填，用户ID
  "startDate": "string",   // 必填，YYYY-MM-DD
  "remark": "string",      // 可选
  "attachments": ["string"] // 可选，文件ID列表
}
```

**响应：**
```json
{
  "code": 0,
  "message": "保存成功"
}
```

**错误码：**

| code | 说明 |
|------|------|
| 1001 | 名称重复 |
| 1002 | 关联订单不存在 |
| 9999 | 系统错误 |
