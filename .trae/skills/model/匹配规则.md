# PRD 模板匹配规则

> 本文件供 AI 在生成 PRD 时自动参照匹配模板

---

## 匹配决策树

```
需求描述
   │
   ├── 是管理后台/B端系统？
   │      ├── 列表页 → 管理后台/01-列表页PRD
   │      ├── 详情页 → 管理后台/02-详情页PRD
   │      ├── 新增/编辑表单 → 管理后台/03-新增编辑页PRD
   │      ├── 数据看板/统计 → 管理后台/04-看板统计页PRD
   │      ├── 审批流程/多级流转 → 管理后台/05-工作流审批页PRD
   │      └── 全局搜索 → 管理后台/06-搜索页PRD
   │
   └── 是用户端/C端产品？
          ├── 首页/入口页 → 用户端/01-首页PRD
          ├── 内容/商品列表 → 用户端/02-列表页PRD
          ├── 详情页 → 用户端/03-详情页PRD
          ├── 表单填写（地址/反馈/登记） → 用户端/04-表单页PRD
          ├── 个人中心/账户管理 → 用户端/05-用户中心PRD
          └── 登录/注册/第三方登录 → 用户端/06-登录注册页PRD
```

---

## 复杂度判断

| 判断维度 | 简单 | 复杂 |
|----------|------|------|
| 页面数量 | 1-2个 | 3个以上 |
| 字段数量 | ≤10个 | >10个 |
| 角色数量 | 1个角色 | 多角色 |
| 系统对接 | 无 | 有上下游系统 |
| 流程 | 单步 | 多步/分支/循环 |

**简单** → 直接用对应模板  
**复杂** → 对应模板 + 通用/完整版PRD.md 补充概述/架构内容

---

## 生成 PRD 的标准流程

1. **提取关键信息**
   - 端：管理后台 / 用户端
   - 页面类型：列表 / 详情 / 表单 / 看板 / 审批 / 搜索 / 首页 / 用户中心 / 登录注册
   - 涉及功能点
   - 复杂度

2. **匹配模板**
   - 按决策树找到对应模板文件
   - 读取模板内容作为结构框架

3. **填充内容**
   - 按模板章节逐项填写
   - 字段、接口、交互必须明确
   - 验收标准前置思考

4. **输出交付**
   - 输出完整 PRD 文档
   - 标注模板来源
