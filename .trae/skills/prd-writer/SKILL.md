# PRD Writer

This skill helps generate comprehensive product requirement documents (PRDs) based on user requirements, prototypes, or existing project information.

## When to Use

Invoke this skill when:
- User asks to create a PRD for a feature or project
- User wants to generate a PRD based on a prototype
- User needs to document product requirements for a new feature
- User requests a structured PRD for any software project

## How to Use

1. Gather information about the project or feature:
   - Project name and purpose
   - Core functionalities
   - User requirements
   - Design prototypes (if available)
   - Technical constraints

2. Provide clear instructions on what should be included in the PRD

3. The skill will generate a comprehensive PRD following industry-standard structure

## PRD Structure

The generated PRD typically includes:

1. **Revision History**
2. **Background**
3. **Project Goals**
4. **Terminology Definitions**
5. **Functional Requirements**
6. **Non-Functional Requirements**
7. **Page Structure**
8. **Core Process**
9. **Interaction Design**
10. **Test Points**
11. **Scope Limitations**
12. **Risk Assessment**
13. **Dependencies**

## Examples

### Example 1: Creating a PRD for a project details page

**Input:**
"Create a PRD for a project details page that includes basic information, service content, and project list sections."

**Output:**
A comprehensive PRD covering all aspects of the project details page, including functionality, design, and implementation requirements.

### Example 2: Generating a PRD from a prototype

**Input:**
"Generate a PRD based on the prototype for the user dashboard page."

**Output:**
A detailed PRD that documents all features and interactions shown in the prototype, with clear requirements and specifications.

## Best Practices

- Provide as much detail as possible about the project or feature
- Include any design prototypes or wireframes if available
- Specify any technical constraints or dependencies
- Clearly define the scope and limitations of the project
- Mention any specific requirements or preferences for the PRD structure

This skill ensures that all PRDs are comprehensive, well-structured, and follow industry best practices, making them valuable documents for development teams, designers, and stakeholders.
