# 尤迈会诊系统需求文档

## 1. 系统概述

尤迈会诊系统是一个医疗会诊服务管理平台，用于管理服务申请、工单处理、项目管理等功能。

## 2. 功能模块

### 2.1 项目管理模块

#### 2.1.1 项目集详情页

##### 2.1.1.1 页面基础信息

| 字段   | 内容          |
| ---- | ----------- |
| 页面名称 | 项目集详情页      |
| 路由   | `/`         |
| 入口说明 | 项目中心 → 项目管理 |
| 使用角色 | 管理员、项目负责人   |

##### 2.1.1.2 页面目标

管理员和项目负责人通过此页面查看项目集的基本信息、服务内容和项目清单，管理项目集的配置和分配。

##### 2.1.1.3 功能结构

```
┌───────────────────────────────────────────────────────────────────────────────┐
│  项目标题 [返回] [消息推送]                                                    │
├───────────────────────────────────────────────────────────────────────────────┤
│  基础信息                                                                   │
│  [编辑] 开始时间/结束时间/项目负责人/客户名称/分配方式/合同/备注             │
├───────────────────────────────────────────────────────────────────────────────┤
│  服务内容                                                                   │
│  服务名称 | 服务数量 | 已发起数量 | 已完成数量 | 完成率 | 限制策略 | 操作  │
├───────────────────────────────────────────────────────────────────────────────┤
│  项目清单                                                                   │
│  [服务分配情况] [新建项目]                                                  │
│  项目名称 | 项目周期 | 进度 | 状态 | 项目经理 | 操作                         │
└───────────────────────────────────────────────────────────────────────────────┘
```

##### 2.1.1.4 基础信息

| 字段    | 类型    | 说明                        |
| ----- | ----- | ------------------------- |
| 开始时间  | 日期    | 项目集的开始时间，格式：YYYY-MM-DD    |
| 结束时间  | 日期    | 项目集的结束时间，格式：YYYY-MM-DD    |
| 项目负责人 | 文本    | 项目集的负责人，如：张明远             |
| 客户名称  | 文本    | 项目集的客户名称，如：阿斯利康投资（中国）有限公司 |
| 分配方式  | 文本    | 项目集的分配方式，如：关联合同           |
| 合同    | 文本/链接 | 项目集关联的合同，可点击跳转到合同详情页      |
| 备注    | 文本    | 项目集的备注信息，如：AZ公益项目         |

**交互说明：**

- 点击编辑按钮 → 打开右侧编辑抽屉
- 点击合同链接 → 跳转到合同详情页

##### 2.1.1.5 服务内容

| 字段    | 类型  | 说明                       |
| ----- | --- | ------------------------ |
| 服务名称  | 文本  | 服务的名称，如：专家案例教学、大病会诊、远程门诊 |
| 服务数量  | 数字  | 服务的总数量，如：1000、500、30     |
| 已发起数量 | 数字  | 已使用服务发起的工单量，如：900、400、15 |
| 已完成数量 | 数字  | 服务的已完成状态数量，如：800、300、10  |
| 完成率   | 百分比 | 服务的完成比例，计算公式：已完成数量/服务数量  |
| 限制策略  | 文本  | 包括专家限制、患者限制、科室限制         |
| 操作    | 按钮  | 查看服务详情                   |

**交互说明：**

- 点击查看按钮 → 跳转到服务详情页
- 点击科室限制中的查看链接 → 弹出科室限制详情弹窗

##### 2.1.1.6 项目清单

| 字段   | 类型  | 说明                                   |
| ---- | --- | ------------------------------------ |
| 项目名称 | 文本  | 项目的名称，如：健康察布查尔2025、健康习水2025          |
| 项目周期 | 文本  | 项目的开始和结束时间，如：2026-01-01 至 2026-12-31 |
| 进度   | 百分比 | 项目的完成进度                              |
| 状态   | 标签  | 项目的状态，如：进行中、已暂停、已归档                  |
| 项目经理 | 文本  | 项目的负责人，如：张经理、李经理                     |
| 操作   | 按钮组 | 查看、更多                                |

**交互说明：**

- 点击查看按钮 → 跳转到项目详情页
- 点击更多按钮 → 弹出操作菜单（编辑、服务限制配置、暂停、恢复、结束）
- 点击服务分配情况按钮 → 弹出数量分配弹窗
- 点击新建项目按钮 → 打开新增项目页面

#### 2.2.3 费用支付步骤 - 选择项目权益

##### 2.2.3.1 权益选择规则

**选择项目权益按钮：**

- 当支付方式选择"项目权益抵扣"时，显示"选择项目权益"按钮
- 点击"选择项目权益"按钮 → 弹出"选择项目权益"弹窗

**已选择权益信息：**

- 选择权益后，弹窗关闭，显示已选择的权益信息卡片
- 权益信息卡片包含：项目名称、项目状态、会诊数量（剩余/总数）、会诊科室数量（剩余/总数）
- 显示"修改项目权益"按钮，点击重新打开选择弹窗

**权益统计规则：**

| 字段 | 计算规则 |
|------|----------|
| 会诊数量-剩余 | 用户可发起的总服务数量 - 使用该权益发起的总量（剔除已取消状态工单） |
| 会诊科室数量-剩余 | 该科室可发起的总服务数量 - 已经发起的数量 |
| 已经发起的数量 | 1. 分诊科室不为空时，分诊科室 = 当前拟申请科室的数量<br>2. 分诊科室为空时，拟申请会诊科室 = 当前拟申请会诊科室的数量 |

**会诊科室选择逻辑：**

- 下拉列表中，根据项目配置中科室限制规则，仅允许选择的科室可选，其他科室显示但不可选
- 选择科室后，显示提示文案："该科室剩余会诊数量：{剩余数量}/{总数量}次"
- 会诊科室数量取值规则：未分诊取"拟申请会诊科室"，已分诊取"会诊科室"字段，数量取两个字段之和

**会诊专家选择逻辑：**

- 下拉列表中，根据项目配置中专家限制规则，仅允许选择的专家可选，其他专家显示但不可选
- 选择专家后，显示提示文案："该项目该专家剩余会诊数量：{剩余数量}/{总数量}次"

**分诊提交校验规则：**

1. 会诊科室校验：
   - 检查所选科室的剩余数量是否为0
   - 如果剩余数量为0，显示提示："会诊科室无剩余数量，不可提交"
   - 阻止提交操作

2. 会诊专家校验：
   - 检查所选专家的剩余数量是否为0
   - 如果剩余数量为0，显示提示："该专家无剩余数量，不可提交"
   - 阻止提交操作

**分诊交互说明：**

- 点击"分诊"按钮 → 打开分诊弹窗
- 选择会诊科室 → 显示科室剩余数量提示
- 选择会诊专家 → 显示专家剩余数量提示
- 点击"确定"按钮 → 校验通过后提交分诊信息
- 点击"取消"按钮 → 关闭弹窗
