# PRD 模板库使用指南

## 模板分类

```
PRD模板库/
├── B端/
│   ├── B端功能迭代PRD.md      # 适用：已有功能的优化改造
│   ├── B端新模块PRD.md        # 适用：从0到1建设新模块
│   ├── B端业务流程PRD.md      # 适用：复杂审批/状态流转场景
│   └── B端数据产品PRD.md      # 适用：报表、BI、数据产品
├── C端/
│   ├── C端功能优化PRD.md      # 适用：用户侧App功能体验优化
│   ├── C端新功能PRD.md        # 适用：用户侧新功能
│   └── C端运营活动PRD.md      # 适用：短期运营活动
└── 通用/
    ├── 轻量版PRD.md          # 适用：简单需求，1-2页搞定
    └── 完整版PRD.md          # 适用：复杂项目，完整评审追溯
```

## 模板选用规则

| 需求特征 | 推荐模板 |
|----------|----------|
| 面向医院/企业内部用户的功能 | B端功能迭代 / B端新模块 |
| 有复杂审批流程、状态配置 | B端业务流程 |
| 涉及指标、报表、数据分析 | B端数据产品 |
| 用户端App/小程序/H5功能优化 | C端功能优化 |
| 用户端新功能上线 | C端新功能 |
| 节日/运营/短期活动 | C端运营活动 |
| 需求简单，参与人数少 | 轻量版PRD |
| 复杂项目，需多方协同 | 完整版PRD |

## 使用方法

1. **复制模板**：复制对应模板到新文件
2. **补充文档信息**：填写标题、版本、作者、日期
3. **按章节编写**：按模板结构逐章填写
4. **关联原型**：在原型链接处附上链接
5. **评审记录**：评审后补充变更记录

## 模板选用建议

- **宁可选大不用小**：如果不确定复杂度，优先选更完整的模板
- **敏捷迭代原则**：B端复杂项目可先用轻量版做快速评审，再补充细节
- **模板可定制**：模板结构只是参考，可根据实际情况调整章节
