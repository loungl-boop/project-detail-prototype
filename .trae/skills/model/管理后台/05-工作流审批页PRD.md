## 3. 状态机定义

| 状态 | 状态值 | 说明 | 可执行操作 |
|------|--------|------|-----------|
| 待提交 | draft | 草稿，未正式发起 | 提交 |
| 审批中 | pending | 流转中 | 审批通过 / 拒绝 / 退回 |
| 已通过 | approved | 审批完成 | - |
| 已拒绝 | rejected | 被拒绝 | 查看 |
| 已撤回 | withdrawn | 申请人撤回 | - |
| 已退回 | returned | 被退回修改 | 重新提交 |

## 4. 审批节点详情

### 4.1 节点1：申请人提交

**处理角色：** 申请人

**提交字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 申请标题 | 文本 | ✓ | 2-100字符 |
| 申请内容 | 多行文本 | ✓ | 说明申请理由 |
| 附件 | 文件 | - | 支持jpg/pdf，单文件≤20MB |
| 审批人 | 人员选择 | ✓ | 指定或根据规则自动分配 |

**提交流程：**
1. 填写表单 → 点击「提交」
2. 系统校验 → 通过后进入审批流
3. 通知当前审批节点负责人

### 4.2 节点2：[审批角色A]

**处理角色：** [审批角色A名称]

**审批操作：**

| 操作 | 按钮 | 结果 |
|------|------|------|
| 通过 | 主按钮 | 流转到下一节点 |
| 拒绝 | 危险按钮 | 流程终止，状态改为「已拒绝」 |
| 退回 | 次按钮 | 退回给申请人修改 |

**审批意见（必填）：**
| 字段 | 类型 | 规则 |
|------|------|------|
| 审批意见 | 多行文本 | 拒绝/退回时必填，1-500字符 |
| 批注 | 多行文本 | 可选 |

**加签/转交：**
- [ ] 支持加签（会签）
- [ ] 支持转交他人审批

## 8. 接口契约

### 8.1 提交申请

**接口路径：** `POST /api/workflow/submit`

### 8.2 审批操作

**接口路径：** `POST /api/workflow/approve`

```json
{
  "id": "string",
  "action": "approve | reject | return",  // 通过/拒绝/退回
  "comment": "string",
  "nextAssignee": "string"  // 退回时必填
}
```

### 8.3 审批记录

**接口路径：** `GET /api/workflow/:id/history`
