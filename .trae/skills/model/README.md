## 模板匹配规则

### Step 1：判断端

| 端 | 适用模板 |
|----|----------|
| 管理后台（B端内部系统） | 管理后台/ 下所有模板 |
| 用户端（C端，面向用户） | 用户端/ 下所有模板 |

### Step 2：判断页面类型

| 页面类型 | 管理后台 | 用户端 |
|----------|----------|--------|
| 列表 + 筛选 + 分页 | 01-列表页 | 02-列表页 |
| 单条信息完整展示 | 02-详情页 | 03-详情页 |
| 表单录入/编辑 | 03-新增编辑页 | 04-表单页 |
| 数据汇总/图表/看板 | 04-看板统计页 | - |
| 审批流程/状态机 | 05-工作流审批页 | - |
| 搜索 + 结果 | 06-搜索页 | - |
| 首页/入口 | - | 01-首页 |
| 个人账户管理 | - | 05-用户中心 |
| 登录/注册 | - | 06-登录注册页 |

### Step 3：判断复杂度

| 复杂度 | 选择模板 |
|--------|----------|
| 简单功能点（1-2个页面） | 轻量版PRD.md |
| 复杂需求（多模块/多角色/多系统） | 完整版PRD.md |

## 使用流程

### 方式一：AI 自动匹配生成

向 AI 描述需求时，包含以下信息，AI 会自动匹配合适模板并生成 PRD：

```
[端] 管理后台 / 用户端
[页面类型] 列表页 / 详情页 / 表单页 / 看板页 / 审批页 / 搜索页 / 首页 / 用户中心 / 登录注册
[复杂度] 简单 / 复杂
[核心功能描述] ...
```

AI 会：
1. 根据端 + 页面类型匹配模板
2. 按模板结构填充内容
3. 输出完整 PRD 文档

### 方式二：手动复制模板

1. 根据需求找到对应模板文件
2. 复制到新文档，重命名为需求名称
3. 按模板章节填充内容
4. 根据实际复杂度决定是否需要完整版模板补充

## 模板填写原则

### 必填项（技术开发必需）

- 路由地址
- 字段清单（字段名、类型、必填、规则）
- 接口契约（请求参数、响应结构）
- 交互说明（点击/跳转/反馈）
- 验收标准

### 选填项（按需补充）

- 用户路径图（复杂流程时）
- 状态流转图（审批类需求）
- 权限设计（多角色系统）
- 非功能性要求（性能/安全）

### 精简原则

- 简单需求用轻量版，不要过度文档化
- 字段清单不必面面俱到，核心技术字段写清楚即可
- 通用字段（创建时间、更新时间）除非业务需要可不写
